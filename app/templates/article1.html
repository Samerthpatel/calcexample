What is pylint?

Pylint is a source-code, bug and quality checker for the Python programming language. It is named following a common convention in Python of a "py" prefix, and a nod to the C programming lint program. It follows the style recommended by PEP 8, the Python style guide.

Pylint example code

Step 1

Pylint can be installed to your project by using the pip install pylint command in the terminal. Since I already have it installed it says requirements already satiafied.

Step 2

Pycharm also fetures a pylint plugin that can be easily installed. To do this go to prefrences then click plugins. Search pylint and install it. Restart pycharm after installing.

Step 3

To implement pylint into your python project set up the Exec path for the plugin so that Pycharm can execute Pylint every time user needs to inspect the code. For that, go to Settings/Preferences, click Other Settings, and then Pylint. Enter the PATH environment variable there and you are good to go.

Step 3

Lets see how it works. Download the example code from the link provided at the start of the Toturial. When running the code we see pylint shows 23 conventions in the file that need to be fixed. For this code, you can see there are many display convention issues. At the lower left, you might see 4 symbols (red, yellow, blue, red ), these symbols tell the user about “Errors”, “Warnings”, “Display Conventions”, and “Refactoring” issues in the code.

Step 4

Pylint describes these errors so you know exactly whats wrong. Try improving the code and fixing some of the pylint errors to see if some of the conventions go away.

Step 5

The pylint.rc file can also be added to the project to change how pylint performs and errors can be disabled by using "#" on top of the errored code and disabling the error.

Terms and definitions

Factory Factory method is a creational design pattern which solves the problem of creating product objects without specifying their concrete classes. Factory Method defines a method, which should be used for creating objects instead of direct constructor call ( new operator).
Instantiation In programming, instantiation is the creation of a real instance or particular realization of an abstraction or template such as a class of objects or a computer process.
Object A unique instance of a data structure that is defined by its class. An object comprises both data members (class variables and instance variables) and methods.
Class A Python class is like an outline for creating a new object.
Namespace A namespace is a system that has a unique name for each and every object in Python.
Constructor Image result for Constructor python A constructor is a special kind of method that Python calls when it instantiates an object using the definitions found in your class.
Fixture Fixtures define the steps and data that constitute the arrange phase of a test
Type hint Type hints may be built-in classes (including those defined in standard library or third-party extension modules), abstract base classes.
Type cast In Python, Type Casting is a process in which we convert a literal of one type to another.
Unit test Unit testing is a software testing method by which individual units of source code are put under various tests to determine whether they are fit for use.
Static When we declare a variable inside a class but outside any method, it is called as class or static variable in python.
Class method A class method is a method which is bound to the class and not the object of the class.
Facade Facade is a structural design pattern that provides a simplified (but limited) interface to a complex system of classes, library or framework.
Method A method in python is somewhat similar to a function, except it is associated with object/classes.
Property The property() method in Python provides an interface to instance attributes
Encapsulation Encapsulation in Python is the process of wrapping up variables and methods into a single entity
Inheritance Inheritance allows us to define a class that inherits all the methods and properties from another class.
Polymorphism Polymorphism in python is used for a common function name that can be used for different types.
Abstraction Abstraction in Python is the process of hiding the real implementation of an application from the user and emphasizing only on how to use the application.
